Procedure to Create and View a Website on the Production Server

  1. Clone your respository to make sure everything goes into your github respository.Use this command-> git clone --config="http.proxy=http://uname:pword@192.168.2.1:8080" http://url-to-your-repo
  2. Then use the command cd(insert respository name) to make sure everything goes into the wanted respository
  3. To create a new website in your respository, use the command gedit (filename.html) & . The & is so that you can use terminal and gedit at the same time.
  4. Then, you can edit this html file however you want as long as the coding is valid.
  5. To view it on your development server, use the command python -m SimpleHTTPServer and then open a new tab on google chrome and type in localhost:8000/(filename.html)
  6. To make sure your code is valid, you have to validate it using this website https://validator.w3.org/
  7. Fix any errors that the validator points out and validate it again. Make sure there are no errors.
  8. Now you can view it on the production server. Use the command sftp sftpusername@cmis-hsteacher . This will allow you to transfer files to a specific production server.
  9. Then it will ask you "Are you sure you want to continue connecting (yes/no)?. Type in yes and hit enter
  10. It will ask you for your password. Type it in and hit enter.
  11. Use the command cd public_html to make sure that you're in the right folder in both yours and the production servers computer.
  12. To upload a file to the production sever, use the command put (filenameyouwanttoupload.html)
  13. Now if you want to view the file you just uploaded on the production server, type in on the search bar http://cmis-hsteacher/~(sftpusername)/(uploadedfilename).
  14. If you want to exit from sftp on your terminal, use the command, Ctrl+D.
